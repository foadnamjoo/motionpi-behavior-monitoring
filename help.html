How to use the report

1. Choose data sources

Under Data collection sources, check the types of data you want in the report:

  • Locations — GPS/location records
  • ENMO — Activity/motion (accelerometer) records
  • Surveys — Survey responses
  • Logs — App log entries (and log-event columns such as disconnects, survey expired, PA/EMA notifications)
  • Batteries — Battery level records for wristbands and phone (adds battery alert columns and Red flag when selected)

Only selected sources appear as columns. Red flag, wristband MACs, phone ID, and battery alert columns are included automatically when relevant; you control their visibility with Show/hide columns after the report runs (see below).

2. Set the time range

Choose Preset (e.g. Last 24 hours, Last 7 days) or Custom range and pick start and end date/time. The report only includes data within this window.

3. Choose chart type

Under Chart type, pick how the activity summary is drawn:

  • Stacked bar (default) — One bar per participant with all collections stacked. You can click a legend item to show or hide that layer in the same chart.
  • Bar chart — One panel per collection (location, ENMO, surveys, etc.), each with its own scale.
  • Horizontal bar — Participants on the Y-axis; useful when there are many participants.

4. Run the report

Click Run report. The app queries the database and may take from a few seconds up to several minutes. When it finishes, the Results section shows, in order: a quick summary line, the activity chart, then Show/hide columns, then the table.

5. Results: chart first, then table

Activity summary (chart) appears first. For Stacked bar: click a label in the legend to add or remove that collection from the chart—hidden layers disappear from the plot and from the legend’s drawn list (no strikethrough). The chart updates immediately. This lets you focus on specific data types (e.g. only location and ENMO) without re-running the report.

Show/hide columns sits right below the chart and above the table. Only columns that exist in your report are listed. Check a box to show that column in the table; uncheck to hide it. The table and CSV export both use only the columns you leave checked.

Table: You can drag a column header to move that column left or right. Drop it between two columns to reorder. This helps compare columns side by side. participantID is always the first column. Hover over any header to see a short explanation (tooltip).

Use Download CSV to export the table (only visible columns and in the order you see). Use Download chart to save the chart as a PNG: for the stacked bar, the image shows exactly what you see on screen—only the layers that are currently visible, and the legend lists only those (no crossed-out items).

Understanding log events

When Logs is selected, the report uses the userlogs collection. Each log record has an eventName, timestamp, participantID, and often phoneID, wristbandMAC, and saltLakeCityTime. The report shows total log count plus counts for key events:

  • Wristband connectivity: connect-wristband-to-app, disconnect-wristband-to-app
  • Survey / PA-EMA: PA_EMA_new-notification (notifications delivered to phone), Activity Survey Expired, PA_denied, morning_survey_trigger, afternoon_survey_trigger, evening_survey_trigger (daily timed EMA)
  • Data collection: Data Collection Enabled, Data Collection Disabled
  • Battery: low_battery_wristband_ + MAC address
  • Dominant hand: Dominant hand change to Left, Dominant hand change to Right

Table columns derived from these include PA/EMA notifications arrived (count of PA_EMA_new-notification), daily timed EMA (sum of morning, afternoon, and evening survey triggers), wristband disconnects, data collection disabled, activity survey expired, and PA survey denied.

Monitoring participant compliance

Use the report and log columns to spot possible misperformance or non-compliance:

  • Red flag count — High or repeated 1h+ low wristband battery (9AM–10PM) suggests the wristband is not being charged.
  • Battery alert counts — Many “below 20%” or “below 10%” records indicate charging is being ignored.
  • Little or no data — Rows with 0 or very low counts (yellow in the table) may indicate a non-collector or device issues.
  • Wristband disconnects — Frequent disconnects can mean pairing problems or the participant removing the device.
  • Data collection disabled — Count of times collection was turned off; high count may reflect intentional disabling.
  • Activity survey expired — Surveys that were not answered within 30 minutes; high count may indicate low engagement.
  • PA survey denied — Participant answered “No” to the physical-activity prompt; useful for engagement patterns.
  • PA/EMA notifications arrived — How many survey notifications were delivered to the phone; compare with submitted survey count to see response gaps.
  • Daily timed EMA — Count of morning, afternoon, and evening survey triggers; reflects how often the app prompted the participant.

For deeper analysis (e.g. by exact event name or time window), query userlogs in MongoDB Compass or another client.

Tip: Red-highlighted rows in the table indicate participants who had at least one “red flag” (wristband battery low for 1 hour or more between 9 AM and 10 PM). The table legend and column tooltips explain the rest.

Metrics for publication

After you run a report, the Study summary panel appears below the quick scan. It provides numbers you can use in the Methods or Results section of a paper. Use Download summary (CSV) or Download summary (JSON) to export a file with each metric, its value, and a short description.

What each metric means

n_participants
Number of participants with at least one record in the selected time window. Use in Methods: “Data from N participants were included.”
date_range
The report time window (e.g. “last 7 days” or “2026-02-01 00:00 to 2026-02-08 23:59 MST”). Use in Methods to state the analysis period.
response_rate_mean / response_rate_sd
Mean and standard deviation of per-participant EMA/survey response rate (submitted surveys ÷ triggers, e.g. daily timed EMA). Only participants with at least one trigger are included. Use in Results: “Response rate was M ± SD (mean ± SD per participant).”
response_rate_overall
Overall response rate: total submitted surveys ÷ total triggers in the window. Use in Results for a single aggregate compliance figure.
n_with_red_flag / pct_with_red_flag
Number and percent of participants with ≥1 red flag (wristband battery low for 1h+ between 9AM–10PM). Use in Results to describe data quality or charging behavior.
n_no_or_minimal_data / pct_no_or_minimal_data
Number and percent of participants with no location, ENMO, or survey records in the window. Use in Methods/Results to describe exclusion or missing data.
mean_* / sd_* (per collection)
Mean and SD of record counts per participant for each data collection (location, ENMO, surveys, logs, batteries). Use in Results to describe data volume or adherence (e.g. “Participants had a mean of M (SD = S) location records per day”).

Glossary — terms used on screen

Report parameters

Data collection sources
Which databases (locations, ENMO, surveys, logs, batteries) to include. Only selected sources appear as columns. Batteries adds battery alert columns and Red flag; Logs adds log-event columns.
Chart type
How the activity summary is drawn: Stacked bar (default; one bar per participant, click legend to show/hide layers), Bar chart (one panel per collection), or Horizontal bar (participants on Y-axis).
Time range / Preset / Custom range
The period of data to include. Preset uses fixed windows (e.g. last 7 days). Custom lets you pick exact start and end date and time.

Results: chart and legend

Activity summary (chart)
The chart appears first in results. For Stacked bar: each legend item is one collection (location, ENMO, surveys, etc.). Click a legend item to hide that layer; click again to show it. The chart redraws with only the visible layers. Hidden items are removed from the legend in the on-screen chart.
Legend (stacked bar)
Labels at the top of the stacked bar chart. Click to toggle that collection on or off. When you Download chart, the PNG shows only the layers you left visible and the legend lists only those (no strikethrough).

Show/hide columns and table

Show/hide columns
Row of checkboxes below the chart and above the table. Only columns that exist in your report are listed. Check = column visible in the table; uncheck = column hidden. Table and CSV export use only the visible columns and their current order.
Drag to reorder columns
Grab a column header and drag it left or right. Drop between two columns to reorder. Helps compare columns. participantID stays first.

Table columns

participantID
Unique identifier for each participant. Always the first column.
Red flag (1h low wristband 9AM–10PM)
Number of times the participant had a wristband battery below 20% for at least 1 hour between 9 AM and 10 PM. Rows with count > 0 are highlighted in red.
left wristband MAC / right wristband MAC
Hardware address of the wristband device. Shown when battery data is included; toggle visibility via Show/hide columns.
phone ID
Identifier of the participant’s phone (last 4 characters). Toggle via Show/hide columns.
location / ENMO / submitted survey / logs / batteries
Counts of records in the selected time range for that participant.
PA/EMA notifications arrived
Count of PA_EMA_new-notification events (survey notifications delivered to the phone).
daily timed EMA
Count of morning, afternoon, and evening survey triggers (morning_survey_trigger, afternoon_survey_trigger, evening_survey_trigger) in the time range.
wristband disconnects / data collection disabled / activity survey expired / PA survey denied
Counts of the corresponding log events. See Understanding log events.
Battery alert columns
Left/right wristband below 20% and phone below 10% alert counts. Visibility can be toggled in Show/hide columns.

Downloads

Download CSV
Exports the table using only the columns that are currently visible and in the order you see (including any drag-to-reorder).
Download chart
Saves the activity summary as a PNG. For the stacked bar, the image matches what you see: only the layers you left visible, and the legend lists only those (no crossed-out items). For bar and horizontal bar, saves the static chart image.
Study summary / Summary (CSV)
The study summary panel shows N participants, date range, response rate (mean ± SD and overall), N (%) with red flag, N (%) with no/minimal data, and mean (SD) records per participant per collection. Each statistic includes a short formula describing how it is calculated. Summary (CSV) downloads these metrics with descriptions for use in papers. See Metrics for publication above.

Quick scan and table legend

Quick scan
Summary line: number of participants, red-flagged, with little/no data, and with battery alerts.
How to read (table legend)
Explains row colors: 0 or empty (possible non-collector), Red flag (wristband low 1h+ 9AM–10PM), Battery alerts. Also notes that you can drag column headers to reorder.
